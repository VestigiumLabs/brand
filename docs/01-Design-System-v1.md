# Préface

Le Design System de VestigiumLabs définit les règles graphiques officielles du laboratoire.

Il constitue la référence unique pour toute interface, documentation, application, site web ou support de communication développé au sein de VestigiumLabs.

Son objectif n'est pas uniquement esthétique.

Il garantit :

- une identité visuelle cohérente ;
- une meilleure expérience utilisateur ;
- une implémentation plus simple ;
- une maintenance facilitée ;
- une évolution maîtrisée.

Chaque composant, chaque couleur et chaque règle décrits dans ce document participent à la construction d'une identité commune.

---

# 1. Philosophie

VestigiumLabs est un laboratoire de recherche.

L'identité graphique doit refléter cette réalité.

Elle ne cherche pas à impressionner.

Elle cherche à inspirer :

- la confiance ;
- la rigueur ;
- la précision ;
- la stabilité.

L'utilisateur doit percevoir immédiatement qu'il consulte un laboratoire technique et non une plateforme commerciale.

Le design est au service du contenu.

Jamais l'inverse.

---

# 2. Les valeurs graphiques

Toutes les interfaces suivent cinq principes.

## 2.1 Simplicité

Chaque élément possède une fonction.

Aucun composant n'est décoratif.

Chaque pixel doit être justifié.

---

## 2.2 Lisibilité

Les contenus techniques sont prioritaires.

Le contraste est élevé.

Les textes restent faciles à lire.

La hiérarchie visuelle est immédiatement compréhensible.

---

## 2.3 Cohérence

Les composants utilisent toujours :

- les mêmes couleurs ;
- les mêmes espacements ;
- les mêmes rayons ;
- les mêmes comportements.

Un utilisateur doit reconnaître immédiatement une interface VestigiumLabs.

---

## 2.4 Pérennité

Les tendances graphiques évoluent.

Le Design System doit rester pertinent dans plusieurs années.

Les choix privilégient donc :

- la sobriété ;
- les standards du Web ;
- les technologies éprouvées.

---

## 2.5 Accessibilité

Les interfaces doivent rester utilisables par le plus grand nombre.

Le Design System applique les principes WCAG :

- contrastes suffisants ;
- tailles de texte adaptées ;
- états visibles ;
- navigation claire.

---

# 5. Le logo

Le logo officiel constitue l'élément central de l'identité.

Il ne doit jamais être modifié.

Le "V" est l'élément de reconnaissance principal.

L'empreinte digitale représente l'investigation numérique.

Les effets visuels du logo officiel font partie intégrante de son identité et doivent être conservés.

---

# Règles d'utilisation

Toujours conserver :

- les proportions ;
- les couleurs ;
- les ombres d'origine ;
- les effets métalliques.

Interdictions :

- étirer le logo ;
- changer ses couleurs ;
- appliquer un filtre ;
- modifier la typographie ;
- retirer l'empreinte.

---

# Zone de protection

Autour du logo, laisser une zone libre équivalente à la hauteur du "V".

Aucun texte ni composant ne doit pénétrer dans cette zone.

---

# Taille minimale

Écran :

48 px

Impression :

15 mm

En dessous de cette taille, utiliser une version simplifiée (à créer ultérieurement si nécessaire).

---

# 6. La bannière

La bannière officielle représente l'univers graphique du laboratoire.

Elle est utilisée :

- sur la page d'accueil ;
- comme couverture GitHub ;
- dans la documentation ;
- sur les réseaux professionnels.

Elle ne doit jamais être découpée ni utilisée comme texture répétitive.

# 12. Ombres

Une seule famille d'ombres est utilisée.

L'objectif est de créer de la profondeur sans attirer l'attention.

Les ombres fortes sont interdites.

---

# 13. Iconographie

Style :

outline

épaisseur uniforme

angles légèrement arrondis

monochrome

Les icônes utilisées doivent respecter les caractéristiques suivantes :

- style outline ;
- épaisseur uniforme ;
- monochromes ;
- angles légèrement arrondis.

La bibliothèque utilisée est laissée au choix de l'implémentation.

---

# 14. Images

Les illustrations privilégient :

- les schémas ;
- les diagrammes ;
- les captures documentées.

Éviter les banques d'images génériques.

---

≤ 767 px

---

# 16. Accessibilité

Contraste minimum AA.

Navigation clavier.

Focus visible.

Zones cliquables ≥ 44 px.

Jamais de texte sur image sans contraste suffisant.

---

# 17. Évolution du Design System

Le présent document est la référence officielle.

Toute évolution doit :

- être documentée ;
- être motivée ;
- préserver la cohérence générale ;
- être validée avant intégration.
- Le Design System évolue uniquement lorsqu'un besoin concret apparaît.

---

# Conclusion

Le Design System de VestigiumLabs n'a pas vocation à suivre les modes.

Il a vocation à durer.

Chaque composant doit transmettre les valeurs du laboratoire :

**Comprendre. Documenter. Transmettre.**
